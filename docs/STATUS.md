## 明确未完成与延期

| 项目 | 状态 | 原因 |
|---|---|---|
| Apple 公证与 stapling | 阻断 | 当前发布环境没有已验证的 notarytool 凭据链 |
| quarantine Gatekeeper 干净安装 | 阻断 | 必须在公证后对真实下载包执行 |
| 干净 macOS 14 / 第二台 Mac / Intel-Rosetta | 待验证 | 历史本机验证不能替代独立系统验收 |
| 通知 TCC、URL 冷热启动、Launch at Login | 待验证 | 自动探针不替代真实系统数据库和生命周期 |
| VoiceOver 与 Full Keyboard Access 全流程 | 待验证 | 自动语义检查已覆盖，人工导航尚未闭环 |
| 长休息完整状态机 | 延期至 3.3 | 触发频率和迁移语义尚未进入规范 |
| quality 评分功能 | 延期 | 当前 `0` 表示未采集，不伪造历史评分 |
| 导入/导出 UI | 延期至 3.3 | Core API 已存在，安全交互规格尚未完成 |
| schema v2 与遗留字段删除 | 延期 | v3.2.x 保持旧 JSON 可读 |
| 第三语言与 `.xcstrings` | 延期 | 不属于当前公开源码门禁 |

## 状态更新规则

1. 不得把“本机通过”写成“远端 CI 通过”，也不得把 runner/Billing 失败记为代码失败或成功。
2. 不得把 Developer ID 签名写成 Apple 公证或 Gatekeeper-ready。
3. 新增公开声明前，必须能指向源代码、自动测试、发布 manifest 或人工验收记录。
4. 公开二进制状态只能在 [OPEN_SOURCE_LAUNCH.md](OPEN_SOURCE_LAUNCH.md) 的全部门禁完成后升级。
